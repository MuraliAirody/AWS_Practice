# Amazon S3 – Java Interview Notes

## 1. What is Amazon S3?

Amazon S3 (Simple Storage Service) is an object storage service provided by AWS.

- Stores data as **objects**
- Inside **buckets**
- Globally unique bucket names
- 99.999999999% (11 9’s) durability
- Strong read-after-write consistency

---

## 2. Core Terminology

### Bucket
A container for objects.
- Region specific
- Globally unique name
- Cannot be renamed

### Object
A file stored in S3.
Consists of:
- Key (file path)
- Value (file content)
- Metadata
- Version ID (if enabled)

### Key
The full path of an object.
Example:

reports/2026/march/sales.csv


Note: S3 does NOT have real folders — folders are part of the key.

---

## 3. S3 Storage Classes

1. S3 Standard – Frequent access
2. S3 Standard-IA – Infrequent access
3. S3 One Zone-IA – Single AZ
4. S3 Intelligent-Tiering – Auto tiering
5. S3 Glacier – Archive
6. S3 Glacier Deep Archive – Cheapest, slow retrieval

---

## 4. Security in S3

S3 permissions are controlled by:

- IAM Policies (Identity-based)
- Bucket Policies (Resource-based)
- ACL (Legacy)
- Block Public Access (Enabled by default)

Encryption Types:
- SSE-S3
- SSE-KMS
- SSE-C

---

## 5. Important Interview Concepts

### Why are bucket names globally unique?
Because S3 buckets are accessed via DNS:

https://bucket-name.s3.amazonaws.com


---

### Difference: IAM Policy vs Bucket Policy

IAM Policy:
- Attached to user/role
- Controls what identity can access

Bucket Policy:
- Attached to bucket
- Controls who can access bucket

---

### 403 vs 404 in S3

- No permission → 403 Forbidden
- Object does not exist → 404 Not Found

---

### What is Versioning?

- Keeps multiple versions of objects
- Protects from accidental delete
- Cannot be permanently disabled (only suspended)

## 7. Create S3 Client
```java
S3Client s3Client = S3Client.builder()
        .region(Region.AP_SOUTH_1)
        .build();
```
## 8. Create Bucket
```java
CreateBucketRequest request = CreateBucketRequest.builder()
        .bucket("murali-interview-bucket-2026")
        .build();

s3Client.createBucket(request);
```
## 9. Upload File
```java
PutObjectRequest putRequest = PutObjectRequest.builder()
        .bucket("murali-interview-bucket-2026")
        .key("test.txt")
        .build();
s3Client.putObject(putRequest,
                   RequestBody.fromFile(Paths.get("C:/files/test.txt")));

```

## 10. Download File
```java
GetObjectRequest getRequest = GetObjectRequest.builder()
        .bucket("murali-interview-bucket-2026")
        .key("test.txt")
        .build();

s3Client.getObject(getRequest,
        Paths.get("C:/files/downloaded-test.txt"));

```
## 11. Delete Object
```java
DeleteObjectRequest deleteRequest = DeleteObjectRequest.builder()
        .bucket("murali-interview-bucket-2026")
        .key("test.txt")
        .build();

s3Client.deleteObject(deleteRequest);

```
## 12. List Objects
```java
ListObjectsV2Request listRequest = ListObjectsV2Request.builder()
        .bucket("murali-interview-bucket-2026")
        .build();

ListObjectsV2Response response = s3Client.listObjectsV2(listRequest);

        System.out.println(obj.key()));

```
## 13. Pre-Signed URL (Very Important Interview Topic)

Used when:

- Backend should not expose AWS credentials

- Client uploads directly to S3
```java

S3Presigner presigner = S3Presigner.create();

PutObjectRequest objectRequest = PutObjectRequest.builder()
        .bucket("murali-interview-bucket-2026")
        .key("upload.txt")
        .build();

PutObjectPresignRequest presignRequest =
        PutObjectPresignRequest.builder()
                .signatureDuration(Duration.ofMinutes(10))
                .putObjectRequest(objectRequest)
                .build();

PresignedPutObjectRequest presignedRequest =
        presigner.presignPutObject(presignRequest);

String url = presignedRequest.url().toString();
System.out.println(url);
```

## 15. Best Practices

- Enable versioning

- Enable lifecycle rules

- Use SSE-KMS for sensitive data

- Never make bucket public unnecessarily

- Use least privilege IAM
