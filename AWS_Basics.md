# AWS Basic Information

###  1. What is AWS?

Amazon Web Services (AWS) is a cloud computing platform provided by Amazon Web Services, a subsidiary of Amazon.

It provides on-demand cloud services such as:

- Compute power

- Storage

- Databases

- Networking

- Security

- Analytics

- Machine Learning

AWS follows a pay-as-you-go pricing model.

### 2. Cloud Computing Models

#### IaaS (Infrastructure as a Service)

- Virtual machines

- Storage

- Networking

Example: EC2

#### PaaS (Platform as a Service)

- Runtime environment

- Middleware

- Managed services

Example: Elastic Beanstalk

#### SaaS (Software as a Service)

Fully managed software applications

Example: Amazon WorkMail

### 3. AWS Global Infrastructure

AWS infrastructure consists of:

Regions

A Region is a physical geographic area.
Example:

- US East (N. Virginia)

- Mumbai (ap-south-1)

#### Availability Zones (AZ)

- One or more discrete data centers within a region.

- Highly available and fault-tolerant.

#### Edge Locations

- Used for CDN (Content Delivery Network).

Example: CloudFront

### 4. Core AWS Services
   #### 4.1 Compute

- Amazon EC2 – Virtual servers

- AWS Lambda – Serverless compute

- Amazon ECS – Container orchestration

- Amazon EKS – Managed Kubernetes

#### 4.2 Storage

- Amazon S3 – Object storage

- Amazon EBS – Block storage

- Amazon EFS – File storage

#### 4.3 Databases

- Amazon RDS – Managed relational DB

- Amazon DynamoDB – NoSQL database

- Amazon Aurora – High-performance relational DB

#### 4.4 Networking

- Amazon VPC – Virtual network

- Elastic Load Balancing – Distributes traffic

- Amazon Route 53 – DNS service

- Amazon CloudFront – CDN

#### 4.5 Security

- AWS IAM – Access control

- AWS KMS – Encryption key management

- AWS Shield – DDoS protection

- AWS WAF – Application firewall

### 5. Pricing Models

- Pay-as-you-go

- Reserved Instances

- Savings Plans

- Spot Instances (up to 90% cheaper)

### 6. AWS Free Tier

- AWS offers a Free Tier for 12 months which includes:

- 750 hours of EC2 per month

- 5GB S3 storage

- Limited RDS usage

### 7. Shared Responsibility Model

- AWS Security is divided into:

- Security of the Cloud (AWS Responsibility)

- Hardware

- Global infrastructure

- Data centers

- Networking

- Security in the Cloud (Customer Responsibility)

- OS patches

- Application security

- IAM configuration

- Data encryption

### 8. Common Use Cases

- Web hosting

- Microservices architecture

- Data analytics

- Backup & disaster recovery

- Machine learning

- DevOps & CI/CD
